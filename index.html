RSA Algorithm

One of the first public-key cryptosystems, RSA (Rivest-Shamir-Adleman), is frequently used for safe data transfer.The encryption key in a public-key cryptosystem is separate from the decryption key, which is kept confidential (private). A user of RSA generates and disseminates a public key using two significant prime integers and an auxiliary value. It is a mystery what the prime numbers are. Anyone can encrypt messages using the public key, but only someone who is familiar with the prime numbers can decode them.

The "factoring problem," which is the practical challenge of factoring the product of two huge prime numbers, is the foundation for RSA security. The RSA issue is the process of deciphering RSA encryption. It is debatable if it is equally challenging as the factoring issue. If a large enough key is used, the system cannot be beaten using any disclosed techniques.

The RSA algorithm is a somewhat slow one. It is not frequently used to directly encrypt user data as a result. Shared keys for symmetric-key cryptography are often sent via RSA, and these keys are then utilised for mass encryption and decryption.

Key Generation

We must first create the two big prime numbers p and q. The product of these primes should be substantially greater than the message we wish to encrypt, and they should be nearly equal in length.

Any primality testing procedure, such as the Miller-Rabin test, can be used to create the primes. We can calculate their product, n = p*q, which will be the modulus for our RSA system, after we know the two prime numbers.

The next step is to select an integer e such that 1<e<phi(n) and gcd(e, phi(n)) = 1, where phi(n) = (p-1)*(q-1) is the totient function of Euler. The exponent of the public key will be this value of e.

We must identify an integer d such that d*e = 1 (mod phi(n)) in order to calculate the private key exponent d. The extended Euclidean method can be utilised for this. Both our public and private keys are (n, e), respectively.

RSA Encryption

A message m must be transformed into an integer between 0 and n-1 in order to be encrypted. A reversible encoding method, such as ASCII or UTF-8, can be used for this.

We calculate the ciphertext c as c = me (mod n) after we obtain the message's integer representation. Using modular exponentiation methods, such as binary exponentiation, this may be accomplished effectively.

RSA Decryption

We calculate the plaintext m as m = cd(mod n) in order to decipher the ciphertext c. Again, we can accomplish this quickly by using modular exponentiation techniques.
